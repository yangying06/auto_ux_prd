# 快速任务 260528-k8w 总结

**任务：** 修复导入后 PRD 分析进度重复且最终失败
**日期：** 2026-05-28
**状态：** 已完成

## 根因

- 前端预置步骤是“正在识别顶层文档包目录...”，后端心跳是“正在识别顶层文档包目录（已等待 X 秒，AI 正在分析原文）”；归一化没有去掉省略号，导致同一阶段被追加成两条进度。
- 顶层拆分和分支拆分共用详细文档包 schema，模型在目录阶段和分支阶段都可能长篇输出并触发 `max_tokens`，最终工具输入被截断成 `{}` 或空数组。
- 分支展开把完整 PRD 反复投给每个并发分支，输入过重；并发分支的心跳还会互相覆盖 `currentStep`。

## 完成内容

- 顶层阶段改成轻量目录 schema：L1 固定 `docPath=null`，字段有长度上限，只做目录规划。
- 分支阶段增加节点数与字段长度上限，并要求每个分支至少返回 1 个可导出的叶子文档包。
- 顶层和分支空结果会触发一次 AI 重试；`max_tokens` 截断会返回更明确的错误。
- 分支展开改为按 L1 的 `extractedFrom` 截取相关 Markdown 章节，不再每个分支吞完整 PRD。
- 并发分支进度聚合为“正在展开文档包分支”，心跳只在 session 仍为 running 时更新，避免错误态和进度列表被其他分支覆盖。
- 前端进度归一化去掉尾部省略号/句点，修复同一阶段重复显示。
