# 快速任务 260528-l8p 总结

**任务：** 修复 AI 顶层拆分返回空导致 PRD 分析失败  
**日期：** 2026-05-28  
**状态：** 已完成

## 根因

诊断接口复现发现，模型可以返回真实顶层文档包，但两个策略过严：

- 模板过滤把所有 `OUTLINE-*` ID 都当成本地模板删除；如果模型自己用了类似 ID，会误删真实结果。
- 顶层节点即使已经有 `docPath`，只要类型是 `module` 就还会被强制展开；独立顶层文档被二次展开后可能返回空分支，导致整个任务失败。

## 完成内容

- 模板过滤改为只识别明确模板文案：`原文标题「...」下的内容`、`本地标题骨架`、`本地兜底节点` 等，不再按 ID 前缀误删。
- 顶层节点只要有 `docPath` 就视为独立文档包，不再强制展开。
- 用最小 PRD 通过 `/api/decompose/start` 和轮询接口复现并验证：最终生成 17 个真实 AI 文档节点，状态为 `done`。
- 单独启动了本地前端和代理服务，确认页面可访问、控制台无错误。
