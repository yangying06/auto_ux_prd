## Visual Hierarchy & Focal Point

**Primary focal point:** The drop zone (dashed border region with 40px upload icon + "Drop your PRD here" heading) is the visual anchor of LandingPage. All other elements are subordinate to it.

Visual reading order on LandingPage:
1. **Drop zone** — primary focal point (largest interactive region, centered, icon + heading)
2. **App title** ("GameUX PromptForge") — secondary identifier, positioned above drop zone
3. **CTA button** — tertiary action trigger, positioned below drop zone

During DecompProgress state, the **progress header** ("Analyzing your PRD...") replaces the drop zone as the focal point, with the step list as the secondary reading zone.

## Interaction Spec

### Upload Flow (UPLD-01, UPLD-02)

```
1. User opens app → LandingPage with UploadCard shown
2. User drags file over drop zone → drag-over state activates
3. User drops file → "Reading file..." state → file validated
   ├─ Invalid file type → inline error, card stays
   └─ Valid .md file → DecompProgress swaps in immediately
4. DecompProgress starts polling /api/decompose/start
5. Steps complete one by one → tree grows in store
6. All steps done (status=done) → auto-navigate to map view (#/)
   └─ Error (status=error) → inline error state in DecompProgress card
```

### Transition: UploadCard → DecompProgress

- **No page reload, no route change** — same DOM container, content swap
- Transition: `transition-opacity duration-300` — fade out UploadCard content, fade in DecompProgress content
- Card shell (bg, border, shadow) stays visible throughout — content inside swaps

### File Picker (click-to-browse)

- `<input type="file" accept=".md,text/markdown" style="display:none">` triggered by button click
- Same validation as drag-drop
- After selection: immediately swap to DecompProgress
